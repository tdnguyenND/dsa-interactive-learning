# Linked List Patterns

Linked list problems in interviews rely on a small set of recurring patterns. Master these techniques and you can solve the vast majority of linked list questions. This page covers the six most important patterns with full implementations.

## Setup: Shared Node Class

Every example on this page uses the same `Node` class and helper functions:

```python
class Node:
    def __init__(self, val, next=None):
        self.val = val
        self.next = next
```

## Pattern 1: Fast and Slow Pointers

The **tortoise and hare** technique uses two pointers moving at different speeds. The slow pointer advances one node per step; the fast pointer advances two. This simple idea solves several problems.

### Find the Middle Node

When fast reaches the end, slow is at the middle.

- For odd-length lists: slow lands on the exact middle
- For even-length lists: slow lands on the second of the two middle nodes

<CodePlayground code={`class Node:
    def __init__(self, val, next=None):
        self.val = val
        self.next = next

def build_list(values):
    dummy = Node(0)
    curr = dummy
    for v in values:
        curr.next = Node(v)
        curr = curr.next
    return dummy.next

def find_middle(head):
    """Return the middle node. O(n) time, O(1) space."""
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    return slow

# Odd length
head = build_list([1, 2, 3, 4, 5])
print("Middle of [1,2,3,4,5]:", find_middle(head).val)

# Even length
head = build_list([1, 2, 3, 4, 5, 6])
print("Middle of [1,2,3,4,5,6]:", find_middle(head).val)`} title="Find Middle Node" />

**Why it works:** When the fast pointer has traveled $n$ nodes, the slow pointer has traveled $n/2$ nodes, which is the middle position.

### Detect a Cycle (Floyd's Algorithm)

If a cycle exists, the fast pointer will eventually meet the slow pointer inside the cycle. If no cycle exists, fast reaches `None`.

<CodePlayground code={`class Node:
    def __init__(self, val, next=None):
        self.val = val
        self.next = next

def has_cycle(head):
    """Returns True if the list has a cycle. O(n) time, O(1) space."""
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            return True
    return False

# List with cycle: 1 -> 2 -> 3 -> 4 -> 5 -> back to node 3
nodes = [Node(i) for i in range(1, 6)]
for i in range(4):
    nodes[i].next = nodes[i + 1]
nodes[4].next = nodes[2]  # creates cycle

print("Cycle detected:", has_cycle(nodes[0]))

# List without cycle
head = Node(1, Node(2, Node(3)))
print("Cycle detected:", has_cycle(head))`} title="Floyd's Cycle Detection" />

### Find the Cycle Start

Once slow and fast meet, reset one pointer to the head. Then move both one step at a time -- they meet at the cycle's entry point.

**Mathematical proof:** Let $d$ be the distance from the head to the cycle start, $k$ be the distance from the cycle start to the meeting point, and $c$ be the cycle length. When they meet:

$$
\text{slow traveled: } d + k, \quad \text{fast traveled: } d + k + mc
$$

Since fast moves twice as far: $2(d + k) = d + k + mc$, so $d + k = mc$, meaning $d = mc - k$. Starting one pointer at the head and one at the meeting point, both moving one step, they converge at the cycle start after $d$ steps.

<CodePlayground code={`class Node:
    def __init__(self, val, next=None):
        self.val = val
        self.next = next

def find_cycle_start(head):
    """Return the node where the cycle begins, or None. O(n) time, O(1) space."""
    slow = fast = head

    # Phase 1: Detect cycle
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            break
    else:
        return None  # no cycle

    # Phase 2: Find entry point
    slow = head
    while slow != fast:
        slow = slow.next
        fast = fast.next
    return slow

# Build cycle: 1 -> 2 -> 3 -> 4 -> 5 -> 6 -> back to 3
nodes = [Node(i) for i in range(1, 7)]
for i in range(5):
    nodes[i].next = nodes[i + 1]
nodes[5].next = nodes[2]  # cycle starts at node with value 3

start = find_cycle_start(nodes[0])
print("Cycle starts at node with value:", start.val)`} title="Find Cycle Start" />

## Pattern 2: Reversal Techniques

Reversing part or all of a linked list appears in many problems. The core idea is the same: redirect `next` pointers using a `prev` variable.

### Reverse Entire List

<CodePlayground code={`class Node:
    def __init__(self, val, next=None):
        self.val = val
        self.next = next

def build_list(values):
    dummy = Node(0)
    curr = dummy
    for v in values:
        curr.next = Node(v)
        curr = curr.next
    return dummy.next

def to_list(head):
    result = []
    while head:
        result.append(head.val)
        head = head.next
    return result

def reverse(head):
    """Reverse entire list. O(n) time, O(1) space."""
    prev = None
    curr = head
    while curr:
        nxt = curr.next
        curr.next = prev
        prev = curr
        curr = nxt
    return prev

head = build_list([1, 2, 3, 4, 5])
print("Original:", to_list(head))
head = reverse(head)
print("Reversed:", to_list(head))`} title="Reverse Entire List" />

### Reverse Between Positions (LeetCode 92)

Reverse only the sublist from position `left` to position `right` (1-indexed). This requires carefully connecting the reversed portion back to the unreversed parts.

<CodePlayground code={`class Node:
    def __init__(self, val, next=None):
        self.val = val
        self.next = next

def build_list(values):
    dummy = Node(0)
    curr = dummy
    for v in values:
        curr.next = Node(v)
        curr = curr.next
    return dummy.next

def to_list(head):
    result = []
    while head:
        result.append(head.val)
        head = head.next
    return result

def reverse_between(head, left, right):
    """Reverse nodes from position left to right (1-indexed).
    O(n) time, O(1) space."""
    if left == right:
        return head

    dummy = Node(0)
    dummy.next = head
    pre = dummy

    # Move pre to the node just before position 'left'
    for _ in range(left - 1):
        pre = pre.next

    # Reverse the sublist using repeated insertion
    curr = pre.next
    for _ in range(right - left):
        nxt = curr.next
        curr.next = nxt.next
        nxt.next = pre.next
        pre.next = nxt

    return dummy.next

head = build_list([1, 2, 3, 4, 5])
print("Original:", to_list(head))

head = reverse_between(head, 2, 4)
print("Reverse positions 2-4:", to_list(head))`} title="Reverse Between Positions" />

The algorithm inserts nodes one at a time right after `pre`, effectively building the reversed portion in place. After each iteration, `curr` stays in place while the node after it leapfrogs to the front of the sublist.

### Reverse Nodes in K-Group (LeetCode 25)

Given a linked list, reverse every group of $k$ consecutive nodes. If the remaining nodes are fewer than $k$, leave them as is. This is a **hard** interview problem.

<CodePlayground code={`class Node:
    def __init__(self, val, next=None):
        self.val = val
        self.next = next

def build_list(values):
    dummy = Node(0)
    curr = dummy
    for v in values:
        curr.next = Node(v)
        curr = curr.next
    return dummy.next

def to_list(head):
    result = []
    while head:
        result.append(head.val)
        head = head.next
    return result

def reverse_k_group(head, k):
    """Reverse nodes in groups of k. O(n) time, O(1) space."""
    # Check if there are at least k nodes remaining
    count = 0
    curr = head
    while curr and count < k:
        curr = curr.next
        count += 1
    if count < k:
        return head  # fewer than k nodes, don't reverse

    # Reverse k nodes
    prev = None
    curr = head
    for _ in range(k):
        nxt = curr.next
        curr.next = prev
        prev = curr
        curr = nxt

    # head is now the tail of this reversed group
    # Recursively reverse the rest and connect
    head.next = reverse_k_group(curr, k)
    return prev  # prev is the new head of this group

head = build_list([1, 2, 3, 4, 5, 6, 7, 8])
print("Original:", to_list(head))

head = reverse_k_group(head, 3)
print("Reverse k=3:", to_list(head))

head = build_list([1, 2, 3, 4, 5])
head = reverse_k_group(head, 2)
print("Reverse k=2 of [1..5]:", to_list(head))`} title="Reverse Nodes in K-Group" />

**Algorithm breakdown:**
1. Check if $k$ nodes remain. If not, return the head unchanged.
2. Reverse the first $k$ nodes using the standard iterative reversal.
3. After reversal, the original `head` becomes the tail of this group. Recursively process the rest of the list and attach it.
4. Return `prev` as the new head of this group.

## Pattern 3: Merge Two Sorted Lists

Given two sorted linked lists, merge them into a single sorted list. This is a building block for merge sort on linked lists.

<CodePlayground code={`class Node:
    def __init__(self, val, next=None):
        self.val = val
        self.next = next

def build_list(values):
    dummy = Node(0)
    curr = dummy
    for v in values:
        curr.next = Node(v)
        curr = curr.next
    return dummy.next

def to_list(head):
    result = []
    while head:
        result.append(head.val)
        head = head.next
    return result

def merge_sorted(l1, l2):
    """Merge two sorted lists into one sorted list.
    O(n + m) time, O(1) space (reuses existing nodes)."""
    dummy = Node(0)
    tail = dummy

    while l1 and l2:
        if l1.val <= l2.val:
            tail.next = l1
            l1 = l1.next
        else:
            tail.next = l2
            l2 = l2.next
        tail = tail.next

    # Attach remaining nodes
    tail.next = l1 if l1 else l2
    return dummy.next

list1 = build_list([1, 3, 5, 7])
list2 = build_list([2, 4, 6, 8, 10])
merged = merge_sorted(list1, list2)
print("Merged:", to_list(merged))

# Edge case: one empty list
list3 = build_list([1, 2, 3])
merged2 = merge_sorted(list3, None)
print("Merge with empty:", to_list(merged2))`} title="Merge Two Sorted Lists" />

The **dummy node** pattern shines here -- it eliminates the need to special-case which list provides the first node. The time complexity is $O(n + m)$ where $n$ and $m$ are the lengths of the two lists.

## Pattern 4: Remove Nth Node From End

To remove the $n$-th node from the end in a single pass, use two pointers separated by $n$ nodes.

<CodePlayground code={`class Node:
    def __init__(self, val, next=None):
        self.val = val
        self.next = next

def build_list(values):
    dummy = Node(0)
    curr = dummy
    for v in values:
        curr.next = Node(v)
        curr = curr.next
    return dummy.next

def to_list(head):
    result = []
    while head:
        result.append(head.val)
        head = head.next
    return result

def remove_nth_from_end(head, n):
    """Remove the nth node from the end. O(L) time, O(1) space.
    Uses a dummy head to handle edge case of removing the first node."""
    dummy = Node(0)
    dummy.next = head
    fast = slow = dummy

    # Advance fast by n+1 steps so the gap is n nodes
    for _ in range(n + 1):
        fast = fast.next

    # Move both until fast reaches the end
    while fast:
        fast = fast.next
        slow = slow.next

    # slow.next is the node to remove
    slow.next = slow.next.next
    return dummy.next

head = build_list([1, 2, 3, 4, 5])
print("Original:", to_list(head))

head = remove_nth_from_end(head, 2)
print("Remove 2nd from end:", to_list(head))

head = build_list([1, 2, 3])
head = remove_nth_from_end(head, 3)
print("Remove 3rd from end of [1,2,3]:", to_list(head))`} title="Remove Nth Node From End" />

**Why $n + 1$ steps?** We want `slow` to stop at the node **before** the target so we can bypass it with `slow.next = slow.next.next`. Advancing fast by $n + 1$ positions ensures the right gap.

## Pattern 5: Palindrome Check

Check if a linked list is a palindrome using $O(1)$ extra space by combining the **find middle** and **reverse** patterns.

<CodePlayground code={`class Node:
    def __init__(self, val, next=None):
        self.val = val
        self.next = next

def build_list(values):
    dummy = Node(0)
    curr = dummy
    for v in values:
        curr.next = Node(v)
        curr = curr.next
    return dummy.next

def is_palindrome(head):
    """Check if linked list is a palindrome.
    O(n) time, O(1) space."""
    if not head or not head.next:
        return True

    # Step 1: Find the middle
    slow = fast = head
    while fast.next and fast.next.next:
        slow = slow.next
        fast = fast.next.next

    # Step 2: Reverse the second half
    prev = None
    curr = slow.next
    while curr:
        nxt = curr.next
        curr.next = prev
        prev = curr
        curr = nxt

    # Step 3: Compare both halves
    first = head
    second = prev
    result = True
    while second:
        if first.val != second.val:
            result = False
            break
        first = first.next
        second = second.next

    # Step 4 (optional): Restore the list
    prev2 = None
    curr = prev
    while curr:
        nxt = curr.next
        curr.next = prev2
        prev2 = curr
        curr = nxt
    slow.next = prev2

    return result

# Test palindromes
for values in [[1,2,3,2,1], [1,2,2,1], [1,2,3], [1]]:
    head = build_list(values)
    print(f"{values} -> palindrome: {is_palindrome(head)}")`} title="Palindrome Linked List" />

**Algorithm:**
1. Find the middle using slow/fast pointers
2. Reverse the second half of the list
3. Compare the first half with the reversed second half
4. (Optional) Restore the list by reversing the second half again

## Pattern 6: Intersection of Two Lists

Find the node where two linked lists merge into one. The key insight: if list A has length $a$ and list B has length $b$, then traversing $a + b$ nodes starting from either head will reach the intersection at the same time.

<CodePlayground code={`class Node:
    def __init__(self, val, next=None):
        self.val = val
        self.next = next

def get_intersection(headA, headB):
    """Find the intersection node of two lists.
    O(n + m) time, O(1) space."""
    if not headA or not headB:
        return None

    a, b = headA, headB

    # When a reaches the end, redirect to headB (and vice versa).
    # After at most a + b steps, they meet at the intersection
    # or both become None (no intersection).
    while a != b:
        a = a.next if a else headB
        b = b.next if b else headA

    return a  # intersection node or None

# Build two lists that share a common tail:
# A: 1 -> 2 -\\
#               -> 6 -> 7 -> 8
# B: 3 -> 4 -> 5 -/
common = Node(6, Node(7, Node(8)))
headA = Node(1, Node(2, common))
headB = Node(3, Node(4, Node(5, common)))

result = get_intersection(headA, headB)
print("Intersection at:", result.val if result else "None")

# No intersection
headC = Node(1, Node(2))
headD = Node(3, Node(4))
result2 = get_intersection(headC, headD)
print("Intersection at:", result2.val if result2 else "None")`} title="Intersection of Two Linked Lists" />

**Why it works:** Pointer `a` travels path $A + B$ and pointer `b` travels path $B + A$. Since both paths have the same total length, they synchronize at the intersection point. If there is no intersection, both become `None` simultaneously.

## Pattern Summary Table

| Pattern | Key Technique | Common Problems |
|---|---|---|
| Fast / Slow Pointers | Two pointers at different speeds | Find middle, detect cycle, cycle start |
| Reversal | Redirect next pointers with prev | Reverse list, reverse between, reverse k-group, palindrome |
| Dummy Head | Sentinel node before real head | Merge sorted, remove nth, any problem with head changes |
| Two-Pointer Gap | Pointers separated by fixed distance | Remove nth from end |
| Merge | Compare-and-attach with dummy | Merge two sorted lists, merge sort |
| Length Alignment | Equalize path lengths | List intersection |

## When to Apply Each Pattern

- **"Find the middle" or "split the list"** -- fast/slow pointers
- **"Is there a cycle?"** -- fast/slow pointers (Floyd's)
- **"Reverse part of the list"** -- in-place reversal with prev/curr/next
- **"Merge sorted lists"** -- two-pointer merge with dummy head
- **"Kth from the end"** -- two-pointer gap technique
- **"Compare first half with second half"** -- find middle + reverse second half (palindrome)
- **"Where do two lists meet?"** -- length alignment with path switching

## Complexity Reference

All patterns on this page run in:

$$
\text{Time: } O(n), \quad \text{Space: } O(1)
$$

This is the gold standard for linked list algorithms -- linear time with constant extra space. The only exception is the recursive reverse-k-group, which uses $O(n/k)$ stack space.

## Key Takeaways

- Most linked list problems are solved by combining just a few core patterns: fast/slow pointers, reversal, dummy heads, and two-pointer gaps.
- The **dummy head** (sentinel) node is your best friend -- it eliminates edge cases when the head might change.
- **Floyd's cycle detection** is the foundation for cycle-related problems. Memorize both the detection and the entry-point finding phases.
- **In-place reversal** is the single most important linked list technique. Practice it until the three-pointer dance (prev, curr, next) is automatic.
- Always consider edge cases: empty list, single node, even vs. odd length, removing the head, and cycles.
- These patterns frequently combine -- for example, the palindrome check uses find-middle, reverse, and compare in sequence.
