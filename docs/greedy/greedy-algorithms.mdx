# Greedy Algorithms

A greedy algorithm makes the **locally optimal choice** at each step, hoping to find the global optimum. Unlike DP, it never reconsiders past choices.

## When does greedy work?

A problem is solvable by greedy if it has:

1. **Greedy choice property**: A locally optimal choice leads to a globally optimal solution
2. **Optimal substructure**: Optimal solution contains optimal solutions to subproblems

**Greedy vs DP**: Greedy makes one choice per step (no going back). DP considers all choices and picks the best. Greedy is faster but only works when the greedy choice property holds.

## Activity selection

Given activities with start/end times, select the maximum number of non-overlapping activities.

**Greedy strategy**: Always pick the activity that **finishes earliest**.

<CodePlayground
  code={`def activity_selection(activities):
    """Select max non-overlapping activities.
    Greedy: sort by end time, pick earliest finishing."""
    # Sort by end time
    activities.sort(key=lambda x: x[1])

    selected = [activities[0]]
    last_end = activities[0][1]

    for start, end in activities[1:]:
        if start >= last_end:
            selected.append((start, end))
            last_end = end

    return selected

## Merge intervals

<CodePlayground
  code={`def merge_intervals(intervals):
    """Merge all overlapping intervals."""
    intervals.sort(key=lambda x: x[0])
    merged = [intervals[0]]

    for start, end in intervals[1:]:
        if start <= merged[-1][1]:
            merged[-1] = (merged[-1][0], max(merged[-1][1], end))
        else:
            merged.append((start, end))

    return merged

intervals = [(1,3), (2,6), (8,10), (15,18)]

## Minimum meeting rooms

<CodePlayground

def min_meeting_rooms(intervals):
    """Minimum rooms needed for all meetings.
    Greedy: sort by start, use min-heap to track end times."""
    if not intervals:
        return 0

    intervals.sort(key=lambda x: x[0])
    heap = []  # min-heap of end times

    for start, end in intervals:
        if heap and heap[0] <= start:
            heapq.heappop(heap)  # reuse a room
        heapq.heappush(heap, end)

    return len(heap)

## Fractional knapsack

Unlike 0/1 knapsack (which needs DP), fractional knapsack allows taking fractions of items — greedy works here.

<CodePlayground
  code={`def fractional_knapsack(capacity, items):
    """items = [(weight, value), ...]
    Greedy: sort by value/weight ratio, take greedily."""
    # Calculate value density and sort
    items_with_ratio = [(v/w, w, v) for w, v in items]
    items_with_ratio.sort(reverse=True)  # highest ratio first

    total_value = 0
    remaining = capacity

    for ratio, weight, value in items_with_ratio:
        if weight <= remaining:
            total_value += value
            remaining -= weight
            print(f"  Take full: weight={weight}, value={value}")
        else:
            fraction = remaining / weight
            total_value += value * fraction
            print(f"  Take {fraction:.2f} of: weight={weight}, value={value*fraction:.1f}")
            break

    return total_value

items = [(10, 60), (20, 100), (30, 120)]  # (weight, value)
capacity = 50
print(f"Capacity: {capacity}")
print(f"Max value: {fractional_knapsack(capacity, items)}")`}
  title="Fractional Knapsack"

## Jump game

<CodePlayground
  code={`def can_jump(nums):
    """Can you reach the last index?
    Greedy: track the farthest reachable position."""
    farthest = 0
    for i, jump in enumerate(nums):
        if i > farthest:
            return False
        farthest = max(farthest, i + jump)
    return True

def min_jumps(nums):
    """Minimum jumps to reach last index.
    Greedy: at each level, find farthest reachable."""
    if len(nums) <= 1:
        return 0
    jumps = 0
    current_end = 0
    farthest = 0
    for i in range(len(nums) - 1):
        farthest = max(farthest, i + nums[i])
        if i == current_end:
            jumps += 1
            current_end = farthest
            if current_end >= len(nums) - 1:
                break
    return jumps

print(can_jump([2, 3, 1, 1, 4]))    # True
print(can_jump([3, 2, 1, 0, 4]))    # False
print(min_jumps([2, 3, 1, 1, 4]))   # 2`}

<CodePlayground

def least_interval(tasks, n):
    """Minimum time units to execute all tasks with cooldown n.
    Greedy: most frequent task determines the frame."""
    freq = Counter(tasks)
    max_freq = max(freq.values())
    max_count = sum(1 for v in freq.values() if v == max_freq)

    # (max_freq - 1) frames of size (n + 1) + last frame
    result = (max_freq - 1) * (n + 1) + max_count
    return max(result, len(tasks))

## Greedy vs DP decision guide

| | Greedy | DP |
|---|--------|-----|
| **Approach** | Pick best local choice | Try all choices |
| **Time** | Usually $O(n \log n)$ | Usually $O(n^2)$ or more |
| **Correctness** | Only if greedy property holds | Always correct |
| **Proof needed** | Yes (exchange argument) | No |
| **Example** | Fractional knapsack | 0/1 knapsack |

---

## Key takeaways

1. Greedy works when **local optimal = global optimal**
2. Sort the input first — most greedy problems start with sorting
3. Interval problems almost always use greedy (sort by end time or start time)
4. If greedy doesn't give the right answer, try DP instead
5. Proving greedy correctness usually requires an **exchange argument**
