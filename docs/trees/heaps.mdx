# Heaps & Priority Queues

A **heap** is a complete binary tree that satisfies the **heap property**: in a min-heap, every parent is smaller than or equal to its children; in a max-heap, every parent is larger. A **priority queue** is the abstract data type that a heap implements -- it supports efficient insertion and extraction of the minimum (or maximum) element.

## Heap Property and Array Representation

Because a heap is a **complete** binary tree (every level is fully filled except possibly the last, which is filled left to right), it maps perfectly to an array with no wasted space:

- The root is at index 0
- For a node at index $i$:
  - **Left child**: $2i + 1$
  - **Right child**: $2i + 2$
  - **Parent**: $\lfloor (i - 1) / 2 \rfloor$

No pointers are needed -- the tree structure is implicit in the array indices.

## Heap Operations

### Insert (Sift Up)

To insert, place the new element at the end of the array (maintaining the complete-tree shape) and **sift it up** by swapping with its parent while the heap property is violated. This takes $O(\log n)$ time since the tree has $\log n$ levels.

### Extract-Min (Sift Down)

To remove the minimum (the root), swap it with the last element, remove the last, and **sift the new root down** by swapping with the smaller child until the heap property is restored. Also $O(\log n)$.

<CodePlayground code={`class MinHeap:
    """Min-heap implementation from scratch."""

    def __init__(self):
        self.data = []

    def __len__(self):
        return len(self.data)

    def peek(self):
        if not self.data:
            raise IndexError("Heap is empty")
        return self.data[0]

    def insert(self, val):
        """Add value and sift up to restore heap property."""
        self.data.append(val)
        self._sift_up(len(self.data) - 1)

    def extract_min(self):
        """Remove and return the minimum element."""
        if not self.data:
            raise IndexError("Heap is empty")
        # Swap root with last element
        self.data[0], self.data[-1] = self.data[-1], self.data[0]
        min_val = self.data.pop()
        if self.data:
            self._sift_down(0)
        return min_val

    def _sift_up(self, idx):
        while idx > 0:
            parent = (idx - 1) // 2
            if self.data[idx] < self.data[parent]:
                self.data[idx], self.data[parent] = self.data[parent], self.data[idx]
                idx = parent
            else:
                break

    def _sift_down(self, idx):
        n = len(self.data)
        while True:
            smallest = idx
            left = 2 * idx + 1
            right = 2 * idx + 2
            if left < n and self.data[left] < self.data[smallest]:
                smallest = left
            if right < n and self.data[right] < self.data[smallest]:
                smallest = right
            if smallest != idx:
                self.data[idx], self.data[smallest] = self.data[smallest], self.data[idx]
                idx = smallest
            else:
                break

## Heapify: Building a Heap in O(n)

Inserting $n$ elements one by one costs $O(n \log n)$. But we can build a heap from an existing array in $O(n)$ by calling sift-down on every non-leaf node, starting from the bottom. This works because roughly half the nodes are leaves (no work needed), a quarter need at most 1 swap, an eighth need at most 2, and so on. The sum $\sum_{h=0}^{\log n} \frac{n}{2^{h+1}} \cdot h = O(n)$.

<CodePlayground code={`def heapify(arr):
    """Convert array into a min-heap in O(n)."""
    n = len(arr)
    # Start from the last non-leaf node
    for i in range(n // 2 - 1, -1, -1):
        sift_down(arr, i, n)
    return arr

def sift_down(arr, idx, n):
    while True:
        smallest = idx
        left = 2 * idx + 1
        right = 2 * idx + 2
        if left < n and arr[left] < arr[smallest]:
            smallest = left
        if right < n and arr[right] < arr[smallest]:
            smallest = right
        if smallest != idx:
            arr[idx], arr[smallest] = arr[smallest], arr[idx]
            idx = smallest
        else:
            break

data = [9, 5, 6, 2, 3, 8, 1, 7, 4]
print(f"Before heapify: {data}")
heapify(data)
print(f"After heapify:  {data}")

n = len(data)
for _ in range(n):
    data[0], data[n-1] = data[n-1], data[0]
    result.append(data[n-1])
    n -= 1
    sift_down(data, 0, n)

## Python's heapq Module

Python provides a min-heap via the `heapq` module. For a max-heap, negate the values.

## Problem: Merge k Sorted Lists

Merge $k$ sorted lists into one sorted list. Push the first element of each list into a min-heap, then repeatedly extract the minimum and push the next element from that list.

<CodePlayground code={`import heapq

def merge_k_sorted(lists):
    """Merge k sorted lists using a min-heap."""
    heap = []
    # Push (value, list_index, element_index)
    for i, lst in enumerate(lists):
        if lst:
            heapq.heappush(heap, (lst[0], i, 0))

    result = []
    while heap:
        val, list_idx, elem_idx = heapq.heappop(heap)
        result.append(val)
        # Push next element from the same list
        if elem_idx + 1 < len(lists[list_idx]):
            next_val = lists[list_idx][elem_idx + 1]
            heapq.heappush(heap, (next_val, list_idx, elem_idx + 1))

    return result

lists = [[1, 4, 5], [1, 3, 4], [2, 6]]
print(f"Input lists: {lists}")
print(f"Merged: {merge_k_sorted(lists)}")`} title="Merge k Sorted Lists" />

## Problem: Find Median from Data Stream

Use two heaps: a max-heap for the smaller half and a min-heap for the larger half. The median is always accessible from the tops of the heaps.

<CodePlayground code={`import heapq

class MedianFinder:
    def __init__(self):
        self.lo = []   # max-heap (negated) for smaller half
        self.hi = []   # min-heap for larger half

    def add_num(self, num):
        # Always push to max-heap first (negated for max-heap behavior)
        heapq.heappush(self.lo, -num)
        # Balance: move largest of lo to hi
        heapq.heappush(self.hi, -heapq.heappop(self.lo))
        # Maintain size: lo can have at most 1 more than hi
        if len(self.hi) > len(self.lo):
            heapq.heappush(self.lo, -heapq.heappop(self.hi))

    def find_median(self):
        if len(self.lo) > len(self.hi):
            return -self.lo[0]
        return (-self.lo[0] + self.hi[0]) / 2

mf = MedianFinder()
stream = [2, 3, 4, 1, 5, 0]
for num in stream:
    mf.add_num(num)
    print(f"Add {num} -> median = {mf.find_median()}")`} title="Find Median from Data Stream" />

## Complexity Summary

| Operation | Time | Notes |
|-----------|------|-------|
| Insert | $O(\log n)$ | Sift up at most $\log n$ levels |
| Extract-min/max | $O(\log n)$ | Sift down at most $\log n$ levels |
| Peek (get min/max) | $O(1)$ | Root is always min/max |
| Heapify | $O(n)$ | Bottom-up sift-down |
| Top $k$ elements | $O(n \log k)$ | Maintain heap of size $k$ |
| Merge $k$ sorted lists (total $N$ elements) | $O(N \log k)$ | Heap of size $k$ |
| Median stream (per insert) | $O(\log n)$ | Two heaps |

## Common Pitfalls

- **Python heapq is min-heap only** -- for max-heap behavior, negate values or use a wrapper
- **Forgetting heapify** -- calling `heappush` $n$ times is $O(n \log n)$; use `heapify` for $O(n)$
- **Comparing tuples** -- when heap elements are tuples, Python compares element by element; ensure the first element is the priority to avoid unexpected orderings
- **Mutating heap elements** -- changing the priority of an element already in the heap does not automatically restore the heap property; use lazy deletion or a separate data structure

## Key Takeaways

- A heap gives $O(\log n)$ insert and extract-min, with $O(1)$ peek -- ideal when you repeatedly need the smallest or largest element
- Array-based representation makes heaps cache-friendly and pointer-free
- **Heapify** builds a heap in $O(n)$, faster than $n$ individual inserts
- Python's `heapq` module covers most use cases; negate values for max-heap
- The **two-heap** pattern (max-heap + min-heap) efficiently tracks the median of a stream
- Heaps are the backbone of Dijkstra's algorithm, Huffman coding, and many scheduling problems
