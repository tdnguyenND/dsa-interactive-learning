# Backtracking

Backtracking explores all possible solutions by building candidates incrementally and **abandoning** (backtracking) a candidate as soon as it cannot lead to a valid solution. It's a systematic way to search through a decision tree with pruning.

## The template

```python
def backtrack(state, choices):
    if is_solution(state):
        result.append(state.copy())
        return

    for choice in choices:
        if is_valid(choice, state):   # prune
            make_choice(choice, state)
            backtrack(state, next_choices)
            undo_choice(choice, state)  # backtrack
```

    """Generate all permutations"""
    result = []

    def backtrack(current, remaining):
        if not remaining:
            result.append(current[:])
            return

        for i in range(len(remaining)):
            current.append(remaining[i])
            backtrack(current, remaining[:i] + remaining[i+1:])

<CodePlayground
  code={`def combine(n, k):
    """Generate all combinations of k numbers from 1..n"""
    result = []

    def backtrack(start, current):
        if len(current) == k:
            result.append(current[:])
            return

        # Prune: not enough numbers left
        for i in range(start, n + 1 - (k - len(current)) + 1):
            current.append(i)
            backtrack(i + 1, current)
            current.pop()

    backtrack(1, [])
    return result

print(f"C(4,2): {combine(4, 2)}")

## N-Queens

Place $n$ queens on an $n \times n$ board so no two queens attack each other.

<CodePlayground
  code={`def solve_n_queens(n):
    result = []
    cols = set()
    diag1 = set()  # row - col
    diag2 = set()  # row + col

    def backtrack(row, board):
        if row == n:
            result.append(["".join(r) for r in board])
            return

        for col in range(n):
            if col in cols or (row-col) in diag1 or (row+col) in diag2:
                continue

            cols.add(col)
            diag1.add(row - col)
            diag2.add(row + col)
            board[row][col] = 'Q'

            backtrack(row + 1, board)

            board[row][col] = '.'
            cols.remove(col)
            diag1.remove(row - col)
            diag2.remove(row + col)

    backtrack(0, board)
    return result

## Word search

<CodePlayground
  code={`def word_search(board, word):
    rows, cols = len(board), len(board[0])

    def backtrack(r, c, idx):
        if idx == len(word):
            return True
        if r < 0 or r >= rows or c < 0 or c >= cols:
            return False
        if board[r][c] != word[idx]:
            return False

        temp = board[r][c]
        board[r][c] = '#'  # mark visited

        for dr, dc in [(0,1),(0,-1),(1,0),(-1,0)]:
            if backtrack(r+dr, c+dc, idx+1):
                return True

        board[r][c] = temp  # unmark
        return False

    for r in range(rows):
        for c in range(cols):
            if backtrack(r, c, 0):
                return True
    return False

board = [
    ['A','B','C','E'],
    ['S','F','C','S'],
    ['A','D','E','E']
]
print(word_search(board, "ABCCED"))   # True
print(word_search(board, "SEE"))      # True

## Complexity of backtracking

Backtracking explores the decision tree, so complexity depends on the branching factor and depth:

| Problem | Time | Space |
|---------|------|-------|
| Subsets | $O(2^n)$ | $O(n)$ |
| Permutations | $O(n!)$ | $O(n)$ |
| N-Queens | $O(n!)$ | $O(n)$ |
| Sudoku | $O(9^{empty})$ | $O(1)$ |

Pruning reduces the actual number of nodes explored, often dramatically.

---

## Key takeaways

1. Backtracking = DFS on the decision tree + **pruning** invalid branches
2. Template: choose → explore → unchoose (undo)
3. Key difference from brute force: pruning skips entire subtrees
4. Common patterns: subsets, permutations, combinations, constraint satisfaction
5. Always think about what to **prune** — it determines practical performance
