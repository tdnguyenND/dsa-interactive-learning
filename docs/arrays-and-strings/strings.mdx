# Strings

Strings are sequences of characters and appear in a vast number of coding problems. Understanding how strings work at the implementation level -- especially immutability -- is crucial for writing efficient solutions.

## String Immutability in Python

In Python, strings are **immutable**: once created, their contents cannot be changed. Every operation that appears to modify a string actually creates a **new string object**.

This has a critical performance implication: repeated string concatenation in a loop is $O(n^2)$, not $O(n)$.

## The StringBuilder Pattern

Because string concatenation creates new objects each time, building a string character by character with `+=` is $O(n^2)$. The fix: collect characters in a **list** and join at the end.

$$\text{Concatenation loop: } O(1 + 2 + 3 + \cdots + n) = O\!\left(\frac{n^2}{2}\right) = O(n^2)$$

$$\text{List join: } O(n) \text{ for appends} + O(n) \text{ for join} = O(n)$$

<CodePlayground code={`import time

def build_with_concat(n):
    """O(n^2) -- creates a new string each iteration."""
    s = ""
    for i in range(n):
        s += str(i % 10)
    return s

def build_with_join(n):
    """O(n) -- appends to list, joins once."""
    parts = []
    for i in range(n):
        parts.append(str(i % 10))
    return "".join(parts)

# Benchmark
for n in [1000, 5000, 20000]:
    start = time.time()
    build_with_concat(n)
    concat_time = time.time() - start

    start = time.time()
    build_with_join(n)
    join_time = time.time() - start

    print(f"n={n:6d}: concat={concat_time:.4f}s, join={join_time:.4f}s, "
          f"ratio={concat_time/max(join_time, 0.0001):.1f}x")`} title="Concatenation vs Join Performance" />

## Common String Operations Complexity

| Operation | Time Complexity | Notes |
|-----------|----------------|-------|
| Access `s[i]` | $O(1)$ | Direct index |
| Slice `s[i:j]` | $O(j - i)$ | Creates new string |
| Concatenate `s + t` | $O(n + m)$ | Creates new string |
| Search `s.find(t)` | $O(n \cdot m)$ | Worst case |
| `in` operator | $O(n \cdot m)$ | Substring search |
| `s.join(list)` | $O(\text{total length})$ | Single allocation |
| `len(s)` | $O(1)$ | Stored as attribute |
| `s == t` | $O(n)$ | Character comparison |

## Essential String Algorithms

### Reverse a String

Since strings are immutable in Python, we convert to a list, reverse, and join. Alternatively, use slicing.

<CodePlayground code={`def reverse_string(s):
    """Reverse using two pointers on a list."""
    chars = list(s)
    left, right = 0, len(chars) - 1
    while left < right:
        chars[left], chars[right] = chars[right], chars[left]
        left += 1
        right -= 1
    return "".join(chars)

def reverse_pythonic(s):
    """Reverse using slice -- idiomatic Python."""
    return s[::-1]

test = "algorithm"
print(f"Original:    '{test}'")
print(f"Two-pointer: '{reverse_string(test)}'")
print(f"Slice:       '{reverse_pythonic(test)}'")

# Reverse words in a sentence
def reverse_words(s):
    """Reverse the order of words."""
    return " ".join(s.split()[::-1])

sentence = "the sky is blue"
print(f"\\nOriginal:  '{sentence}'")
print(f"Reversed:  '{reverse_words(sentence)}'")`} title="String Reversal" />

### Palindrome Check

A palindrome reads the same forwards and backwards. Compare characters from both ends toward the center.

<CodePlayground code={`def is_palindrome(s):
    """Check if string is a palindrome (letters/digits only)."""
    # Clean: lowercase, keep only alphanumeric
    cleaned = [c.lower() for c in s if c.isalnum()]
    left, right = 0, len(cleaned) - 1
    while left < right:
        if cleaned[left] != cleaned[right]:
            return False
        left += 1
        right -= 1
    return True

test_cases = [
    "racecar",
    "A man, a plan, a canal: Panama",
    "hello",
    "Was it a car or a cat I saw?",
    ""
]

for s in test_cases:
    result = is_palindrome(s)
    print(f"  '{s}' -> {result}")`} title="Palindrome Check" />

### Valid Anagram

Two strings are anagrams if they contain exactly the same characters with the same frequencies. Use a frequency counter for $O(n)$ time.

<CodePlayground code={`from collections import Counter

def is_anagram(s, t):
    """Check if t is an anagram of s using frequency counting."""
    if len(s) != len(t):
        return False

    # Method 1: Counter (most Pythonic)
    return Counter(s) == Counter(t)

def is_anagram_manual(s, t):
    """Check using a manual frequency array (interview-friendly)."""
    if len(s) != len(t):
        return False

    freq = [0] * 26  # Assuming lowercase English letters
    for c in s:
        freq[ord(c) - ord('a')] += 1
    for c in t:
        freq[ord(c) - ord('a')] -= 1

    return all(f == 0 for f in freq)

pairs = [
    ("anagram", "nagaram"),
    ("listen", "silent"),
    ("hello", "world"),
    ("rat", "car"),
]

for s, t in pairs:
    r1 = is_anagram(s, t)
    r2 = is_anagram_manual(s, t)
    print(f"  '{s}' vs '{t}': Counter={r1}, Manual={r2}")`} title="Valid Anagram" />

### Longest Common Prefix

Find the longest common prefix among an array of strings. Compare characters column by column.

<CodePlayground code={`def longest_common_prefix(strs):
    """Find longest common prefix by vertical scanning."""
    if not strs:
        return ""

    # Use the shortest string as the upper bound
    min_len = min(len(s) for s in strs)

    prefix = []
    for col in range(min_len):
        char = strs[0][col]
        # Check if all strings have the same char at this position
        if all(s[col] == char for s in strs):
            prefix.append(char)
        else:
            break

    return "".join(prefix)

def lcp_sorting(strs):
    """Alternative: sort and compare only first and last."""
    if not strs:
        return ""
    strs.sort()
    # After sorting, the most different strings are first and last
    first, last = strs[0], strs[-1]
    i = 0
    while i < len(first) and i < len(last) and first[i] == last[i]:
        i += 1
    return first[:i]

tests = [
    ["flower", "flow", "flight"],
    ["dog", "racecar", "car"],
    ["interview", "internal", "internet", "interval"],
    ["a"],
    [],
]

for strs in tests:
    r1 = longest_common_prefix(strs)
    r2 = lcp_sorting(strs[:])  # Copy since sort mutates
    print(f"  {strs}")
    print(f"    Vertical scan: '{r1}', Sorting: '{r2}'\\n")`} title="Longest Common Prefix" />

### Character Frequency Counting

Counting character occurrences is a building block for many string problems: anagrams, permutation checks, and frequency-based puzzles.

<CodePlayground code={`from collections import Counter

def char_frequency(s):
    """Count character frequencies."""
    freq = {}
    for c in s:
        freq[c] = freq.get(c, 0) + 1
    return freq

# Find first non-repeating character
def first_unique_char(s):
    """Return index of first non-repeating character, or -1."""
    freq = Counter(s)
    for i, c in enumerate(s):
        if freq[c] == 1:
            return i
    return -1

## String Matching Algorithms

### Brute Force: $O(n \cdot m)$

For every position in the text, check if the pattern matches starting there.

<CodePlayground code={`def brute_force_search(text, pattern):
    """Find all occurrences of pattern in text. O(n*m) worst case."""
    n, m = len(text), len(pattern)
    positions = []

    for i in range(n - m + 1):
        match = True
        for j in range(m):
            if text[i + j] != pattern[j]:
                match = False
                break
        if match:
            positions.append(i)

    return positions

text = "ABABDABACDABABCABAB"
pattern = "ABAB"
positions = brute_force_search(text, pattern)
print(f"Text:    '{text}'")
print(f"Pattern: '{pattern}'")
print(f"Found at positions: {positions}")

# Visualize matches
for pos in positions:
    print(f"  {'.' * pos}{pattern}{'.' * (len(text) - pos - len(pattern))}")`} title="Brute Force String Matching" />

### KMP Algorithm Overview

The Knuth-Morris-Pratt (KMP) algorithm improves on brute force by using information from partial matches to skip unnecessary comparisons. It preprocesses the pattern into a **failure function** (also called the prefix table) that tells us how far to shift on a mismatch.

Time: $O(n + m)$, Space: $O(m)$ for the failure table.

<CodePlayground code={`def kmp_search(text, pattern):
    """KMP string matching algorithm. O(n + m) time."""
    n, m = len(text), len(pattern)
    if m == 0:
        return []

    # Build failure function (longest proper prefix-suffix)
    lps = [0] * m
    length = 0
    i = 1
    while i < m:
        if pattern[i] == pattern[length]:
            length += 1
            lps[i] = length
            i += 1
        else:
            if length != 0:
                length = lps[length - 1]
            else:
                lps[i] = 0
                i += 1

    print(f"Pattern:  {pattern}")
    print(f"LPS table: {lps}")

    # Search
    positions = []
    i = j = 0  # i for text, j for pattern
    while i < n:
        if text[i] == pattern[j]:
            i += 1
            j += 1
        if j == m:
            positions.append(i - j)
            j = lps[j - 1]
        elif i < n and text[i] != pattern[j]:
            if j != 0:
                j = lps[j - 1]  # Skip using LPS
            else:
                i += 1

    return positions

text = "ABABDABACDABABCABAB"
pattern = "ABAB"
print(f"Text: '{text}'\\n")
positions = kmp_search(text, pattern)
print(f"\\nFound at positions: {positions}")`} title="KMP String Matching" />

## String Complexity Summary

| Algorithm / Operation | Time | Space | Notes |
|----------------------|------|-------|-------|
| Reverse string | $O(n)$ | $O(n)$ | New string in Python |
| Palindrome check | $O(n)$ | $O(n)$ | After cleaning |
| Anagram check (Counter) | $O(n)$ | $O(k)$ | $k$ = alphabet size |
| Longest common prefix | $O(S)$ | $O(1)$ | $S$ = total chars |
| Brute force matching | $O(nm)$ | $O(1)$ | |
| KMP matching | $O(n + m)$ | $O(m)$ | Prefix table |
| String concatenation loop | $O(n^2)$ | $O(n)$ | Immutability cost |
| List join | $O(n)$ | $O(n)$ | Preferred approach |

## Key Takeaways

1. **Strings are immutable in Python.** Every modification creates a new object. Use list-based building and `"".join()` for efficient construction.
2. **Frequency counting** with `Counter` or arrays is the backbone of anagram, permutation, and character-based problems.
3. **Two-pointer palindrome check** is a pattern you will reuse across many problems -- understand it thoroughly.
4. **KMP** achieves $O(n + m)$ string matching through its prefix table, avoiding redundant comparisons. While you may not need to implement it from scratch in interviews, understanding the concept is valuable.
5. **Always consider the hidden cost** of operations like `s += char` in loops or `s.find()` inside loops -- these can silently turn an $O(n)$ algorithm into $O(n^2)$.
