# Shortest Path Algorithms

Finding the shortest path between nodes is one of the most fundamental graph problems. While BFS solves this for unweighted graphs, **weighted graphs** require specialized algorithms. This page covers two essential algorithms: **Dijkstra's** for non-negative weights and **Bellman-Ford** for graphs with negative weights.

## Problem Definition

Given a weighted directed graph $G = (V, E)$ with weight function $w: E \to \mathbb{R}$ and a source vertex $s$, find the shortest-weight path from $s$ to every other vertex. The shortest path weight from $s$ to $v$ is defined as:

$$\delta(s, v) = \min \{ w(p) : p \text{ is a path from } s \text{ to } v \}$$

If no path exists, $\delta(s, v) = \infty$.

## Dijkstra's Algorithm

Dijkstra's is a **greedy** algorithm that works on graphs with **non-negative edge weights**. It maintains a priority queue (min-heap) of vertices ordered by their tentative distance from the source.

### How It Works

1. Initialize $dist[s] = 0$ and $dist[v] = \infty$ for all other vertices.
2. Push $(0, s)$ into a min-heap.
3. While the heap is not empty:
   - Pop the vertex $u$ with minimum distance.
   - For each neighbor $v$ of $u$, if $dist[u] + w(u, v) < dist[v]$, update $dist[v]$ and push $(dist[v], v)$ into the heap.

The key insight is that when a vertex is popped from the heap, its shortest distance is finalized (because all weights are non-negative).

def dijkstra(graph, start):
    """Dijkstra's shortest path algorithm.
    graph: {node: [(neighbor, weight), ...]}
    Returns: distances dict and parent dict for path reconstruction.
    """
    dist = {start: 0}
    parent = {start: None}
    heap = [(0, start)]
    visited = set()

    while heap:
        d, u = heapq.heappop(heap)
        if u in visited:
            continue
        visited.add(u)

        for v, weight in graph.get(u, []):
            new_dist = d + weight
            if new_dist < dist.get(v, float('inf')):
                dist[v] = new_dist
                parent[v] = u
                heapq.heappush(heap, (new_dist, v))

    return dist, parent

## Bellman-Ford Algorithm

Bellman-Ford handles graphs with **negative edge weights** and can detect **negative weight cycles**. It relaxes all edges $|V| - 1$ times, since the shortest path can have at most $|V| - 1$ edges.

### How It Works

1. Initialize $dist[s] = 0$ and $dist[v] = \infty$ for all other vertices.
2. Repeat $|V| - 1$ times: for every edge $(u, v, w)$, if $dist[u] + w < dist[v]$, update $dist[v]$.
3. Perform one more iteration. If any distance still decreases, a **negative cycle** exists.

<CodePlayground code={`def bellman_ford(num_nodes, edges, start):
    """Bellman-Ford shortest path algorithm.
    edges: list of (u, v, weight)
    Returns: (distances, parents, has_negative_cycle)
    """
    dist = [float('inf')] * num_nodes
    parent = [-1] * num_nodes
    dist[start] = 0

    # Relax all edges V-1 times
    for i in range(num_nodes - 1):
        updated = False
        for u, v, w in edges:
            if dist[u] != float('inf') and dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                parent[v] = u
                updated = True
        if not updated:
            break  # Early termination

    # Check for negative weight cycles
    for u, v, w in edges:
        if dist[u] != float('inf') and dist[u] + w < dist[v]:
            return dist, parent, True  # Negative cycle detected

    return dist, parent, False

## Dijkstra vs Bellman-Ford

| Aspect | Dijkstra | Bellman-Ford |
|--------|----------|-------------|
| Time complexity | $O((V + E) \log V)$ with binary heap | $O(V \cdot E)$ |
| Space complexity | $O(V)$ | $O(V)$ |
| Negative weights | Not supported | Supported |
| Negative cycle detection | No | Yes |
| Approach | Greedy (min-heap) | Dynamic programming (relaxation) |
| Typical use | Road networks, GPS | Currency arbitrage, network routing |

## Algorithm Selection Guide

| Scenario | Best Algorithm | Reason |
|----------|---------------|--------|
| Unweighted graph | BFS | $O(V + E)$, simplest |
| Non-negative weights | Dijkstra | $O((V + E) \log V)$, fastest |
| Negative weights possible | Bellman-Ford | Handles negatives, $O(VE)$ |
| Need to detect negative cycles | Bellman-Ford | Extra iteration check |
| All-pairs shortest paths | Floyd-Warshall | $O(V^3)$, simple DP |
| Sparse graph, all-pairs | Run Dijkstra from each vertex | $O(V(V+E) \log V)$ |
| DAG (directed acyclic) | Topological sort + relaxation | $O(V + E)$ |

## Complexity Summary

| Algorithm | Time | Space | Negative Weights | Negative Cycle Detection |
|-----------|------|-------|-----------------|-------------------------|
| BFS (unweighted) | $O(V + E)$ | $O(V)$ | N/A | No |
| Dijkstra (binary heap) | $O((V + E) \log V)$ | $O(V + E)$ | No | No |
| Dijkstra (Fibonacci heap) | $O(V \log V + E)$ | $O(V + E)$ | No | No |
| Bellman-Ford | $O(V \cdot E)$ | $O(V)$ | Yes | Yes |
| Floyd-Warshall | $O(V^3)$ | $O(V^2)$ | Yes | Yes |

## Key Takeaways

- **Dijkstra's algorithm** is the standard choice for shortest paths in graphs with non-negative weights. It runs in $O((V+E) \log V)$ with a binary heap.
- Dijkstra is **greedy**: once a node is popped from the heap, its shortest distance is final. This property breaks with negative edges.
- **Bellman-Ford** handles negative weights by relaxing all edges $|V|-1$ times. It runs in $O(V \cdot E)$.
- Bellman-Ford detects **negative weight cycles** with one additional iteration: if any distance decreases, a negative cycle is reachable from the source.
- For **unweighted graphs**, use BFS. For **DAGs**, use topological sort with single-pass relaxation for $O(V+E)$.
- Always check: are weights non-negative? If yes, use Dijkstra. If no, use Bellman-Ford.
