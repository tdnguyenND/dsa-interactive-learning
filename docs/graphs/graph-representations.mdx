# Graph Representations

Graphs model relationships between objects using vertices (nodes) and edges. This page covers the two primary ways to represent graphs: adjacency lists and adjacency matrices, along with their trade-offs.

## Overview

An adjacency list stores a list of neighbors for each vertex, making it space-efficient for sparse graphs. An adjacency matrix uses a 2D array where entry (i, j) indicates an edge between vertex i and vertex j, providing O(1) edge lookups at the cost of O(V^2) space.

        self.graph[u].append(v)

## Complexity

| Operation         | Adjacency List | Adjacency Matrix |
|-------------------|---------------|-----------------|
| Add Edge          | O(1)          | O(1)            |
| Remove Edge       | O(E)          | O(1)            |
| Check Edge Exists | O(V)          | O(1)            |
| Space             | O(V + E)      | O(V^2)          |
