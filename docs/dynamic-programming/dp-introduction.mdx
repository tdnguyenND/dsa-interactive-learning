# Introduction to Dynamic Programming

Dynamic Programming (DP) solves problems by breaking them into **overlapping subproblems** and storing results to avoid redundant computation. It's one of the most powerful algorithmic techniques.

## When to use DP

A problem is a DP candidate if it has:

1. **Optimal substructure**: The optimal solution contains optimal solutions to subproblems
2. **Overlapping subproblems**: The same subproblems are solved multiple times

If a problem has optimal substructure but **no** overlapping subproblems, use divide and conquer instead (e.g., merge sort).

## Two approaches

### Top-down (Memoization)

Start from the original problem, recurse into subproblems, cache results.

### Bottom-up (Tabulation)

Build the solution from the smallest subproblems up to the original problem using a table.

<CodePlayground
  code={`# Fibonacci: the canonical DP example

# 1. Naive recursion — O(2^n) time
def fib_naive(n):
    if n <= 1:
        return n
    return fib_naive(n - 1) + fib_naive(n - 2)

# 2. Top-down with memoization — O(n) time, O(n) space
def fib_memo(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    memo[n] = fib_memo(n - 1, memo) + fib_memo(n - 2, memo)
    return memo[n]

# 3. Bottom-up tabulation — O(n) time, O(n) space
def fib_tab(n):
    if n <= 1:
        return n
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]

# 4. Space-optimized — O(n) time, O(1) space
def fib_opt(n):
    if n <= 1:
        return n
    prev2, prev1 = 0, 1
    for _ in range(2, n + 1):
        curr = prev1 + prev2
        prev2, prev1 = prev1, curr
    return prev1

# Compare
for n in [10, 30, 50]:
    print(f"fib({n}) = {fib_opt(n)}")
print(f"fib(100) = {fib_opt(100)}")`}

## DP problem-solving framework

1. **Define the state**: What does `dp[i]` (or `dp[i][j]`) represent?
2. **Find the recurrence**: How does `dp[i]` relate to smaller subproblems?
3. **Base cases**: What are the trivially solvable states?
4. **Iteration order**: Ensure dependencies are computed before they're needed
5. **Optimize space** (optional): Often you only need the last 1-2 rows/values

## House robber

<CodePlayground
  code={`def rob(nums):
    """Max money without robbing adjacent houses.
    State: dp[i] = max money considering houses 0..i
    Recurrence: dp[i] = max(dp[i-1], dp[i-2] + nums[i])
    """
    if not nums:
        return 0
    if len(nums) == 1:
        return nums[0]

    prev2, prev1 = 0, 0
    for num in nums:
        curr = max(prev1, prev2 + num)
        prev2, prev1 = prev1, curr
    return prev1

print(rob([1, 2, 3, 1]))      # 4 (rob house 0 and 2)
print(rob([2, 7, 9, 3, 1]))   # 12 (rob house 0, 2, 4)
print(rob([2, 1, 1, 2]))      # 4 (rob house 0 and 3)`}
  title="House Robber"

## Coin change

<CodePlayground
  code={`def coin_change(coins, amount):
    """Minimum coins to make amount.
    State: dp[i] = min coins needed for amount i
    Recurrence: dp[i] = min(dp[i - c] + 1) for each coin c
    """
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0

    for i in range(1, amount + 1):
        for c in coins:
            if c <= i and dp[i - c] + 1 < dp[i]:
                dp[i] = dp[i - c] + 1

    return dp[amount] if dp[amount] != float('inf') else -1

print(coin_change([1, 5, 10, 25], 30))   # 2 (25+5)
print(coin_change([1, 3, 4], 6))          # 2 (3+3)
print(coin_change([2], 3))                # -1`}
  title="Coin Change"
/>

## Longest increasing subsequence

<CodePlayground
  code={`def lis(nums):
    """Length of longest increasing subsequence — O(n^2)
    State: dp[i] = length of LIS ending at index i
    Recurrence: dp[i] = max(dp[j] + 1) for j < i where nums[j] < nums[i]
    """
    if not nums:
        return 0
    n = len(nums)
    dp = [1] * n

    for i in range(1, n):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)

    print(f"dp array: {dp}")
    return max(dp)

arr = [10, 9, 2, 5, 3, 7, 101, 18]
print(f"Array: {arr}")
print(f"LIS length: {lis(arr)}")  # 4: [2, 3, 7, 101]`}

## Top-down vs bottom-up comparison

| | Top-down (Memo) | Bottom-up (Tab) |
|---|----------------|-----------------|
| **Style** | Recursive + cache | Iterative + table |
| **Computes** | Only needed states | All states |
| **Stack** | Recursion depth limit | No stack issues |
| **Debugging** | Harder (recursion) | Easier (loop) |
| **Space opt** | Harder | Easier |

## Common DP patterns

| Pattern | Example | State |
|---------|---------|-------|
| **Linear** | Climbing stairs, house robber | `dp[i]` |
| **Knapsack** | Coin change, 0/1 knapsack | `dp[i][w]` or `dp[w]` |
| **String** | Edit distance, LCS | `dp[i][j]` |
| **Grid** | Unique paths, min path sum | `dp[i][j]` |
| **Interval** | Matrix chain, burst balloons | `dp[i][j]` range |
| **State machine** | Stock problems | `dp[i][state]` |

---

## Key takeaways

1. DP = **overlapping subproblems** + **optimal substructure**
2. Two approaches: **memoization** (top-down) and **tabulation** (bottom-up)
3. Framework: define state → recurrence → base case → iteration order → optimize space
4. Start with brute-force recursion, then add memoization, then convert to tabulation
5. Most DP problems reduce to filling a 1D or 2D table
